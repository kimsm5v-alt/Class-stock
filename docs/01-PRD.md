# Class-Stock PRD (Product Requirements Document)

> **버전:** MVP v1.0
> **작성일:** 2026.03.17
> **스코프:** 파일럿 운영 가능한 최소 기능 범위
> **기술 범위:** React + TypeScript 프론트엔드 + Supabase BaaS
> **용도:** Claude Code 개발 작업 지시서

---

## 1. 제품 개요

### 1.1 한 줄 요약

수업 시간에 핵심 개념을 '주식'처럼 사고파는 학습 게이미피케이션 웹앱.

### 1.2 목표

파일럿 학교 3~5개 학급에서 **"교사가 반복 사용하는가?"**를 검증한다. MVP에서는 기본 모드(수업 끝 1회 거래)만 구현하고, 고급 모드(수시 거래)는 MVP 이후로 미룬다.

### 1.3 사용자

| 사용자 | 디바이스 | 접속 방식 |
|---|---|---|
| **교사** | PC/태블릿 (전자칠판 브라우저) | 이메일 회원가입 → 로그인 |
| **학생** | 태블릿/노트북 브라우저 | 수업 코드(6자리) + 닉네임 입력 (가입 불필요) |

### 1.4 MVP 핵심 제약

- **기본 모드만 구현**: 수업 중 찜하기 + 수업 끝 1회 거래
- **고급 모드(수시 거래 오픈) 제외**: MVP 이후
- **모둠 협력 모드 제외**: MVP 이후
- **누적 포트폴리오/학기 랭킹 제외**: MVP에서는 수업 단위 결과만
- **AI 인사이트 제외**: MVP에서는 단순 통계만 (AI 분석은 이후)

## 5. 데이터 모델 (Supabase)

### 5.1 테이블 구조

```sql
teachers
├── id (uuid, PK)
├── email (text, unique)
├── name (text)
├── created_at (timestamptz)

sessions (= 수업)
├── id (uuid, PK)
├── teacher_id (uuid, FK → teachers)
├── class_name (text)          -- "3학년 2반"
├── subject (text)             -- "과학"
├── unit_name (text)           -- "세포의 에너지"
├── join_code (text, unique)   -- 6자리 수업 코드
├── status (text)              -- 'waiting' | 'active' | 'trading' | 'settling' | 'closed'
├── trade_end_at (timestamptz) -- 거래 오픈 마감 시각 (null이면 거래 비활성)
├── created_at (timestamptz)

stocks (= 종목/키워드)
├── id (uuid, PK)
├── session_id (uuid, FK → sessions)
├── keyword (text)             -- "미토콘드리아"
├── is_hidden (boolean)        -- 히든 종목 여부
├── is_revealed (boolean)      -- 공개 여부 (false면 학생에게 ??? 표시)
├── is_core (boolean)          -- 정산 시 핵심 키워드로 선택되었는지
├── display_order (int)        -- 표시 순서
├── created_at (timestamptz)

students (= 수업 참여 학생, 세션 단위)
├── id (uuid, PK)
├── session_id (uuid, FK → sessions)
├── nickname (text)
├── balance (int, default 10000) -- 투자 가능 잔고
├── created_at (timestamptz)

bookmarks (= 찜하기)
├── id (uuid, PK)
├── student_id (uuid, FK → students)
├── stock_id (uuid, FK → stocks)
├── created_at (timestamptz)
├── UNIQUE(student_id, stock_id)

trades (= 매수/매도 기록)
├── id (uuid, PK)
├── student_id (uuid, FK → students)
├── stock_id (uuid, FK → stocks)
├── type (text)                -- 'buy' | 'sell'
├── amount (int)               -- 투자 포인트
├── created_at (timestamptz)

holdings (= 현재 보유 현황)
├── id (uuid, PK)
├── student_id (uuid, FK → students)
├── stock_id (uuid, FK → stocks)
├── amount (int)               -- 현재 보유 포인트
├── UNIQUE(student_id, stock_id)
```

### 5.2 Realtime 구독 대상

| 테이블 | 이벤트 | 구독자 | 용도 |
|---|---|---|---|
| sessions | UPDATE | 교사 + 학생 | status 변경 감지 (장 개시(수업 시작), 거래 오픈, 장 마감) |
| stocks | UPDATE | 학생 | is_revealed 변경 → 종목 공개 실시간 반영 |
| students | INSERT | 교사 | 학생 접속 실시간 카운트 |
| bookmarks | INSERT/DELETE | 교사 | 찜하기 관심도 실시간 업데이트 |
| holdings | INSERT/UPDATE | 교사 | 매수율 실시간 업데이트 |

## 7. 수업 흐름 (기본 모드)

```
[교사] 수업 개설 → 키워드 입력 → 히든 종목 지정
         ↓
[교사] 수업 대기실 (QR/코드 공유)
         ↓
[학생] 수업 코드 + 닉네임으로 입장 → 대기
         ↓
[교사] "장 개시(수업 시작)" 클릭 → status: 'active'
         ↓
[교사] 종목 하나씩 공개 (is_revealed: true)
[학생] 공개된 종목 확인 + 찜하기
         ↓
[교사] "거래 오픈" 클릭 → status: 'trading', trade_end_at 설정
         ↓
[학생] 카운트다운 보면서 매수/매도
         ↓
[시스템] 거래 오픈 종료 → status: 'settling'
         ↓
[교사] 핵심 키워드 3개 선택 → 정산 실행
         ↓
[학생] 폭등 애니메이션 + 수익률 확인
         ↓
[교사] 학급 리포트 확인 → status: 'closed'
```

## 8. 정산 로직

### 8.1 수익률 계산

```
핵심 종목 보유 시: 투자금 × 2 (100% 수익)
비핵심 종목 보유 시: 투자금 × 0.5 (50% 손실)
미투자 잔고: 그대로 유지
```

### 8.2 예측 정확률

```
정확률 = (핵심 종목 투자액 / 총 투자액) × 100%
```

## 10. 보안 고려사항

### 10.1 RLS (Row Level Security)

- 교사는 자신의 수업만 관리 가능
- 학생은 참여한 수업의 데이터만 접근 가능
- 찜하기/거래는 본인 기록만 생성/수정 가능

### 10.2 수업 코드

- 6자리 영숫자 (충돌 방지)
- 수업 종료 후 재사용 불가 (또는 24시간 후 만료)
