# Class-Stock 요구사항명세서 (SRS)

> **버전:** MVP v1.0
> **작성일:** 2026.03.17
> **용도:** Claude Code 개발 작업 지시서

---

## 1. 시스템 개요

Class-Stock은 교사와 학생이 동일한 수업 세션에 실시간으로 접속하여, 수업 핵심 개념에 대한 투자 게임을 진행하는 웹 애플리케이션이다. 교사는 PC/태블릿 브라우저, 학생은 태블릿/노트북 브라우저로 접속한다.

## 2. 기능 요구사항

### 2.1 인증 및 사용자 관리

| ID | 요구사항 | 상세 |
|---|---|---|
| FR-AUTH-01 | 교사 이메일 회원가입 | Supabase Auth 이메일/비밀번호. 가입 시 이름 입력 필수 |
| FR-AUTH-02 | 교사 로그인/로그아웃 | 이메일/비밀번호 로그인. 세션 유지 (Supabase 토큰) |
| FR-AUTH-03 | 학생 익명 입장 | 수업 코드(6자리) + 닉네임 입력으로 세션 참여. 별도 가입 없음 |
| FR-AUTH-04 | 학생 중복 닉네임 방지 | 동일 세션 내 같은 닉네임 사용 불가 |

### 2.2 수업(세션) 관리

| ID | 요구사항 | 상세 |
|---|---|---|
| FR-SESS-01 | 수업 개설 | 학급명, 과목, 단원명 입력. 수업 코드(6자리) 자동 생성. 영문+숫자 조합, 대소문자 무관 |
| FR-SESS-02 | 키워드 입력 | 최소 3개, 최대 7개. 각 키워드 최대 20자 |
| FR-SESS-03 | 히든 종목 지정 | 입력한 키워드 중 최대 2개를 히든으로 지정 가능 |
| FR-SESS-04 | QR 코드 생성 | 수업 코드 기반 접속 URL을 QR로 렌더링. 화면에 크게 표시 |
| FR-SESS-05 | 수업 상태 관리 | waiting → active → trading → settling → closed 순서로 전이 |
| FR-SESS-06 | 수업 이력 조회 | 교사의 과거 수업 목록. 최신순 정렬. 각 수업의 상태, 날짜, 학급명 표시 |

### 2.3 수업 진행 — 교사

| ID | 요구사항 | 상세 |
|---|---|---|
| FR-TEACH-01 | 학생 접속 실시간 카운트 | 대기실에서 접속 학생 수 실시간 표시 (Supabase Realtime) |
| FR-TEACH-02 | 장 개시(수업 시작) | "장 개시(수업 시작)" 버튼 → session.status를 'active'로 변경. 학생 화면 자동 전환 |
| FR-TEACH-03 | 종목 개별 공개 | 각 종목 옆 "공개" 버튼 → stock.is_revealed = true. 학생에게 실시간 반영 |
| FR-TEACH-04 | 찜하기 관심도 표시 | 종목별 찜한 학생 수를 실시간 표시 (숫자 + 비율) |
| FR-TEACH-05 | 거래 오픈 | "거래 오픈" 버튼 → session.status를 'trading'으로 변경 + trade_end_at 설정 (현재시각 + 거래 시간) |
| FR-TEACH-06 | 거래 시간 설정 | 거래 오픈 전 30초 / 60초 / 90초 / 120초 중 선택. 기본값 60초 |
| FR-TEACH-07 | 거래 자동 마감 | trade_end_at 도달 시 자동으로 거래 종료. 또는 교사가 수동 "거래 마감" 가능 |
| FR-TEACH-08 | 핵심 키워드 선택 | 장 마감 후 전체 종목 중 핵심 키워드 3개 선택 (체크박스) → 정산 실행 |
| FR-TEACH-09 | 정산 실행 | 핵심 종목 보유 학생의 해당 holding amount × 3 (300% 폭등). 비핵심 종목은 amount × 0.5 (50% 하락) |
| FR-TEACH-10 | 실시간 매수율 대시보드 | 거래 중 종목별 매수율(매수 학생 수 / 전체 학생 수) 바 차트. 실시간 갱신 |
| FR-TEACH-11 | 리포트: 수업 한 눈에 | 정산 후 3개 핵심 지표 카드: 학급 평균 정확률(= 핵심 종목 투자 비율 평균), 참여율(= 거래 1회 이상 학생 비율), 총 거래 건수 |
| FR-TEACH-12 | 리포트: 개념별 분석 (인지-확신 갭) | 종목별 찜하기율과 매수율 이중 바 차트. 찜하기율 - 매수율 > 20%p인 종목에 "갭 발생" 태그 표시. 핵심(★)/히든(🔒) 종목 아이콘 구분 |
| FR-TEACH-13 | 리포트: 학급 투자 성향 | 종목별 투자 비율 도넛 차트 + 성향 분류(몰빵형/분산형/양극화형) 배지 + 성향별 맞춤 코멘트. 추가: 종목 공개 순서 vs 매수율 상관관계, 히든 종목 도전율 |
| FR-TEACH-14 | 리포트: 놓친 개념 알림 | 핵심 종목 중 매수율 30% 미만인 종목에 경고 카드 표시. 인지-확신 갭 부가 정보 포함. 해당 없으면 "핵심 개념이 모두 잘 전달되었습니다" 표시 |
| FR-TEACH-15 | 리포트: 수업 추이 | 같은 교사의 과거 수업(최근 5~10회)과 비교하는 정확률/참여율 추이 라인 차트. 직전 수업 대비 변화량 표시. 1회차인 경우 안내 텍스트만 표시 |

### 2.4 수업 진행 — 학생

| ID | 요구사항 | 상세 |
|---|---|---|
| FR-STU-01 | 수업 입장 | 수업 코드 입력 → 유효성 검증 → 닉네임 입력 → students 테이블에 레코드 생성 |
| FR-STU-02 | 대기 화면 | session.status가 'waiting'일 때 대기 화면. "교사가 수업을 시작하면 자동으로 넘어갑니다" |
| FR-STU-03 | 포인트 지급 | 입장 시 balance = 10,000pt |
| FR-STU-04 | 종목 리스트 표시 | 미공개 종목은 "??? (미공개)" 표시. 공개된 종목은 키워드명 표시. is_revealed 변경 실시간 반영 |
| FR-STU-05 | 찜하기 | 공개된 종목 + 히든 종목(???) 모두 찜하기 가능. 토글 방식 (찜/찜 해제). session.status가 'active' 또는 'trading'일 때 가능 |
| FR-STU-06 | 매수 | 거래 오픈 시(session.status = 'trading')에만 활성화. 종목 선택 → 투자 금액 입력 (1,000pt 단위) → 매수. balance 차감, holdings 증가 |
| FR-STU-07 | 매도 | 거래 오픈 시에만 활성화. 보유 종목 선택 → 전액 매도. holdings 감소, balance 복구 |
| FR-STU-08 | 잔고 표시 | 화면 하단에 항상 표시: 투자 가능 잔고 / 투자 완료 금액 |
| FR-STU-09 | 거래 카운트다운 | 거래 오픈 시 남은 시간 카운트다운 표시 (trade_end_at 기준) |
| FR-STU-10 | 정산 결과 표시 | 핵심 종목 폭등 애니메이션 + 히든 종목 정체 공개 + 최종 수익률 표시 |
| FR-STU-11 | 이해도 정확률 | 정산 후 "이해도 예측 정확률" 표시: (핵심 종목에 투자한 포인트 / 총 투자 포인트) × 100 |

## 3. 비기능 요구사항

| ID | 항목 | 요구사항 |
|---|---|---|
| NFR-01 | 실시간성 | 종목 공개, 찜하기, 거래 상태 변경이 **2초 이내**에 모든 클라이언트에 반영 |

| NFR-06 | 로딩 속도 | 초기 로딩 3초 이내 (Vercel CDN 기준) |
| NFR-07 | 에러 처리 | 네트워크 끊김 시 재연결 시도 + 사용자 안내. Supabase Realtime 재구독 |
| NFR-08 | 데이터 정합성 | 매수/매도 시 balance 차감은 서버 사이드(Supabase RPC 또는 Database Function)에서 처리하여 음수 잔고 방지 |

## 4. 제약사항

| 항목 | 제약 |
|---|---|
| 인증 | 학생은 Supabase 익명 인증 또는 세션 기반. 교사만 이메일 가입 |
| 결제 | 없음. 무료 서비스 |
| 개인정보 | 학생의 실명, 학번 등 수집하지 않음. 닉네임만 사용 |
| 동시 수업 | 한 교사가 동시에 여러 수업을 진행하는 경우는 MVP에서 미지원 (1교사 1활성세션) |
| 오프라인 | 미지원. 인터넷 연결 필수 |

## 5. 세션 상태 전이도

```
waiting ──(장 개시(수업 시작))──→ active ──(거래 오픈)──→ trading ──(거래 마감)──→ active
                         │                                                │
                         │                        (장 마감 + 핵심 선택)    │
                         │                                                ▼
                         └──────────────────────────────────────────→ settling ──(정산 완료)──→ closed
```

- `waiting`: 학생 접속 대기 중
- `active`: 수업 진행 중 (찜하기 가능, 거래 불가)
- `trading`: 거래 오픈 (매수/매도 가능)
- `settling`: 교사가 핵심 키워드 선택 → 정산 중
- `closed`: 수업 종료, 결과 열람만 가능
