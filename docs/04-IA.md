## 2. 세션 상태 (Session Status)

```
waiting ──(장 개시(수업 시작))──→ active ──(거래 오픈)──→ trading ──(거래 마감)──→ active
                           │                                               │
                           │                       (장 마감 + 핵심 선택)    │
                           │                                               ▼
                           └─────────────────────────────────────────→ settling ──(정산 완료)──→ closed
```

| 상태 | 설명 | 교사 액션 | 학생 가능 행동 |
|---|---|---|---|
| `waiting` | 학생 접속 대기 중 | QR/코드 공유, "장 개시(수업 시작)" 버튼 대기 | 접속 + 대기만 |
| `active` | 수업 진행 중 | 종목 공개, 대시보드 모니터링 | **찜하기 ☆** (공개된 종목 + 히든 ???) |
| `trading` | 거래 오픈 | 매수율 실시간 모니터링, 수동 마감 가능 | **찜하기 + 매수 + 매도** |
| `settling` | 교사가 핵심 키워드 선택 중 | 핵심 3개 선택 → 정산 실행 | 대기 (자동 전환) |
| `closed` | 수업 종료 | 학급 리포트 열람 | 정산 결과 열람 |

**핵심 포인트: 찜하기는 `active` 상태부터 가능하고, 매수/매도는 `trading` 상태에서만 가능하다.**

## 3. 화면 전이 흐름

### 3.1 교사 플로우

```
SCR-01 Landing
  │
  └─ "교사로 시작" ─→ SCR-02 Auth (회원가입/로그인)
                        │
                        └─ 로그인 성공 ─→ SCR-03 Dashboard (수업 목록)
                                           │
                                           ├─ "새 수업 만들기" ─→ SCR-04 Create (수업 개설)
                                           │                       │
                                           │                       └─ "수업 만들기" ─→ SCR-05 Waiting Room
                                           │                                            │
                                           │                                            └─ "장 개시(수업 시작)" ─→ SCR-06 Live Session
                                           │                                                               │
                                           │                                                               └─ "장 마감 → 정산" ─→ SCR-07 Settlement
                                           │                                                                                       │
                                           │                                                                                       └─ "정산 실행" ─→ SCR-08 Report
                                           │                                                                                                         │
                                           │                                                                                                         └─ 복귀 ─→ SCR-03
                                           │
                                           └─ 종료된 수업 클릭 ─→ SCR-08 Report
```

### 3.2 학생 플로우

```
SCR-01 Landing
  │
  └─ "학생으로 참여" ─→ SCR-09 Join (수업코드 + 닉네임)
                          │
                          └─ 입장 성공 ─→ [세션 상태에 따라 분기]
                                           │
                                           ├─ waiting ─→ SCR-10 Waiting (대기)
                                           │               │
                                           │               └─ status→active 감지 ─→ SCR-11 Live Trading
                                           │
                                           ├─ active ─→ SCR-11 Live Trading
                                           │
                                           └─ trading ─→ SCR-11 Live Trading
                                                          │
                                                          └─ status→closed 감지 ─→ SCR-12 Result
                                                                                     │
                                                                                     └─ "수업 나가기" ─→ SCR-09 Join
```

### 3.3 교사-학생 동기화 포인트

```
교사 화면                              학생 화면
─────────                             ─────────

SCR-05 Waiting Room ◄── 접속 ──── SCR-09 Join
  │ 학생 접속 실시간 카운트               │ 입장 → SCR-10 대기

SCR-06 Live Session ◄── realtime ──→ SCR-11 Live Trading
  │                                      │
  ├─ 종목 공개 ──────────────────→ 종목 이름 실시간 노출
  ├─ 거래 오픈 ──────────────────→ 매수/매도 버튼 활성화
  ├─ 거래 마감 ──────────────────→ 매수/매도 버튼 비활성화
  │                                      │
  ◄── 찜하기 데이터 ─────────────── 찜하기 ☆ 탭
  ◄── 매수/매도 데이터 ──────────── 매수/매도 실행

SCR-07 Settlement ──── 정산 ────→ SCR-12 Result
  │ 핵심 키워드 확정                    │ 폭등 애니메이션 + 수익률
```

## 4. 학생 행동 가능 범위 (상태별)

학생이 각 세션 상태에서 할 수 있는 행동을 명확히 정의한다. 이 표가 프론트엔드 UI 활성/비활성 로직의 기준이 된다.

| 행동 | waiting | active | trading | settling | closed |
|---|---|---|---|---|---|
| 수업 접속 | ✅ | ✅ | ✅ | ❌ | ❌ |
| 종목 리스트 확인 | ❌ | ✅ (공개된 것만) | ✅ | ❌ | ✅ (결과) |
| 찜하기 ☆ | ❌ | **✅** | **✅** | ❌ | ❌ |
| 매수 | ❌ | ❌ | **✅** | ❌ | ❌ |
| 매도 | ❌ | ❌ | **✅** | ❌ | ❌ |
| 잔고 확인 | ❌ | ✅ (10,000pt 고정) | ✅ (변동) | ❌ | ✅ (최종) |
| 수익률 확인 | ❌ | ❌ | ✅ (실시간) | ❌ | ✅ (최종) |
| 정산 결과 확인 | ❌ | ❌ | ❌ | ❌ | ✅ |

## 6. 인증 구조

```
교사:
  Supabase Auth (이메일/비밀번호)
  → 로그인 시 teachers 테이블에서 프로필 조회
  → 세션 토큰으로 인증 상태 유지
  → 보호 라우트: /teacher/* 는 인증 필수

학생:
  가입 없음
  → 수업 코드(6자리) + 닉네임 입력
  → students 테이블에 레코드 생성 (session_id, nickname)
  → student.id를 로컬 상태(Zustand)에 보관
  → 보호 라우트: /session/* 는 student.id 필수
```

## 7. 용어 정의

| 용어 | 설명 | UI 표기 |
|---|---|---|
| 장 개시(수업 시작) | 교사가 수업을 시작하여 학생 화면을 활성화하는 행위. session.status → active | "장 개시(수업 시작)" 버튼 |
| 거래 오픈 | 교사가 학생의 매수/매도를 허용하는 행위. session.status → trading | "거래 오픈" 버튼 |
| 거래 마감 | 거래 오픈을 닫는 행위 (자동 또는 수동). session.status → active | "거래 마감" 버튼 / 타이머 만료 |
| 장 마감 | 수업 전체를 종료하고 정산 단계로 넘어가는 행위 | "장 마감 → 정산" 버튼 |
| 정산 | 교사가 핵심 키워드 3개를 선택하고 포인트를 계산하는 행위. session.status → closed | "정산 실행" 버튼 |
| 종목 공개 | 교사가 ??? 상태의 키워드를 학생에게 공개하는 행위. stock.is_revealed → true | "공개" 버튼 |
| 찜하기 | 학생이 관심 종목을 표시하는 가벼운 인터랙션 (투자 아님) | ☆ 아이콘 토글 |
| 매수 | 학생이 종목에 포인트를 투자하는 행위 (거래 오픈 시에만) | "매수" 버튼 |
| 매도 | 학생이 보유 종목을 전액 환수하는 행위 (거래 오픈 시에만) | "매도" 버튼 |
| 히든 종목 | 교사가 지정한 비공개 종목. 수업 끝까지 ??? 상태 유지 가능 | 🔒 + "HIDDEN" 태그 |
| 인지-확신 갭 | 찜하기율(인지)과 매수율(확신) 간의 차이. 갭이 클수록 학생이 중요성은 알지만 확신하지 못하는 상태 | 리포트 화면 |
| 핵심 종목 | 정산 시 교사가 선택한 정답 키워드. ×3 폭등 적용 | ★ + "핵심" 태그 |

## 8. 리포트 화면 구성 (wireframe v3)

### 8.1 교사 리포트 (SCR-08)

교사가 정산 완료 후 확인하는 학급 분석 화면.

**구성 요소:**
1. **수업 한 눈에** - 학급 평균 정확률, 참여율, 총 거래 건수
2. **개념별 분석** - 찜하기율 vs 매수율 비교 (인지-확신 갭 시각화)
3. **개념 이해도 매트릭스** - 사분면 차트 (X: 찜하기율, Y: 매수율)
4. **학급 투자 성향** - 종목별 투자 비율 분포 + 성향 태그 (양극화형/분산형/쏠림형)
5. **놓친 개념 알림** - 핵심 종목인데 매수율이 낮은 경우 경고
6. **수업 추이** - 최근 수업들의 정확률/참여율 변화

**사분면 해석:**
- 우상단 (찜↑ 매수↑): 잘 이해한 개념
- 좌상단 (찜↑ 매수↓): 인지-확신 갭 (다른 방식으로 재설명 필요)
- 좌하단 (찜↓ 매수↓): 완전히 놓친 개념 (처음부터 다시)
- 우하단 (찜↓ 매수↑): 소수만 확신 (또래 교수법 활용)

### 8.2 학생 결과 (SCR-12)

학생이 정산 완료 후 확인하는 개인 결과 화면.

**구성 요소:**
1. **최종 수업 결과** - 최종 포인트, 변동률, 이해도 예측 정확률
2. **종목별 결과** - 각 종목의 투자 금액과 수익 (핵심 종목 ×3 표시)
3. **히든 종목 공개** - 정체불명이었던 종목의 실제 키워드 공개
4. **수업 나가기** - JoinSession 화면으로 복귀
